Installation Guide

Prerequisities:

  • Installed Java 7 JDK. You should also have IDE, such as Netbeans or Eclipse. In case of Eclipse, the latest version is recommended, so that git/maven are already available there.
  • Installed Apache Maven 3. It may be provided together with certain IDEs
  • Installed Git version control tool It may be provided together with certain IDEs
  • Installed Apache Tomcat 7
  • Installed Virtuoso 6.1.7 (NOT Virtuoso 7)

Installation:

  1. Obtain the source code from GitHub, master branch.
  2. In the downloaded odcs folder, further denoted as {root}, copy {root}/conf/config.propertis to the {home}/.odcs/config.properties, where {home} is your home folder on the system.
  3. Adjust {home}/.odcs/config.properties: Set properties general.workingdir (the directory where the working directory of the executed pipelines is located), module.path (the directory where the data processing units are located). Specify connection to the Virtuoso using virtuoso.* properties
  4. Start Virtuoso Instance, prepare virtuoso database by populating the database with the scripts:
    1. {root}/db/virtuoso/rdbms/schema.sql, which prepares the core tables needed
    2. {root}/db/virtuoso/rdbms/data.sql, which inserts core data processing units and sample data to the database
    Both scripts should be executed by the Virtuoso command utility isql -U {username} -P {password} -S {isqlPort} < {file}
  5. Clean and build the project by running mvn install on the {root} folder. Make sure that Virtuoso is up and running, it is needed for certain tests. If Virtuoso is not running, execute mvn install -DskipTests instead of that
  6. Deploy frontend -- {root}/frontend/target/odcs.war -- on Apache Tomcat and check that the application is running under the context /odcs
  7. Run backend responsible for executing the pipelines. It may be run by executing java -jar backend-{version}.jar. Under Linux system, check that backend has enough rights to work with temporary working directory and other files; if you are not sure, run backend as: sudo java -jar backend-{version}.jar. If you run backend or frontend under a different user than You, please copy {home}/.odcs/config.properties to the appropriate home folder of that user, which might be home folder of root in case of superuser on Linux or C:/.odcs/config.properties in case of system user on Windows (this feature will be adjusted soon).
